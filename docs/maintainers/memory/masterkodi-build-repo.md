As of 2026-07-10, the MasterKodi IL ecosystem moved to a Moran-style automated model in a NEW public repo **github.com/asaf27064/MasterKodi-IL-Build** (created this session). It consolidates ALL addons under `addons/` and is the single source of truth.

**How updates work now (full replace of the old monolithic wizard flow):**
- CI (`.github/workflows/build-and-release.yml`) on every push builds each `addons/<id>/` into a **reproducible** zip (deterministic sha256 via `tools/common.py` — fixed entry order + timestamps), writes `manifest.json` (id/version/channel/sha256/size/url per addon), uploads only *changed* zips to the rolling `addons-latest` release, prunes orphans, commits the manifest.
- Client: wizard `resources/libs/modular_update.py` (wizard bumped to **2.4.0**) reads the manifest, compares installed version+sha, downloads only changed, **verifies sha256**, extracts, `UpdateLocalAddons`. Downgrade-safe (semver compare); same-version-different-sha still updates (for the Gears Hebrew overlay whose addon.xml version stays 2.3.1). Optional skins update only if already installed. Wired into `default.py` check-updates button + `service.py` startup.
- `build.json` marks `channels.optional` = skin.arctic.fuse.3, skin.nimbus.
- Manifest URL clients read: `https://raw.githubusercontent.com/asaf27064/MasterKodi-IL-Build/main/manifest.json`

**EXE/APK auto-build (verified working):**
- `build-apk.yml` (Linux): apktool + base Kodi APK (from our `build-inputs` release) + inject wizard/firstrun/repo+deps, **sign** with keystore. Base script `installers/android/build_kodi_il_apk_v3.py` (made cross-platform: zipalign/apksigner without .exe/.bat on Linux; needs Pillow).
- `build-exe.yml` (Windows): Inno Setup + KodiFiles/portable_data (from `build-inputs`) via `installers/windows/build_exe.py`.
- Both publish to the `installers` release. **Keystore is now a GH secret** (`KEYSTORE_B64`/`KEYSTORE_ALIAS`/`KEYSTORE_PASS`, alias `kodiil`, pass `kodiil123`) — NOT committed (better than the old public-keystore-in-zip). Big base binaries live in the `build-inputs` release.
